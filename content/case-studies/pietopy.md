### The Product Requirements (PRD)

**Objective**: To create a unified, frictionless ecosystem for Python learning that eliminates the need for local setup and fragmented learning tools.

#### Key Product Pillars
- **Zero Configuration**: Leveraging **Pyodide (Wasm)** to provide a full scientific stack (NumPy, Pandas) directly in the browser.
- **AI-First Pedagogy**: "PyBot" for real-time debugging assistance and "Teach Me" mode for personalized curriculum delivery.
- **Project-Centric**: Beyond just syntax—enabling users to build and preview full-stack (Python/HTML/JS) apps in one tab.

---

### Market Research & User Empathy

Interviews with educators revealed that 40% of classroom time is wasted on "environment setup issues" (installing Python, IDEs, libraries).

**Pietopy** solves this by:
1. **Instant Environment**: A single URL provides a high-fidelity IDE.
2. **Contextual Help**: Using AI to bridge the gap between error messages and conceptual understanding.

---

### Technical Architecture

- **Engine**: Pyodide (WebAssembly) for client-side Python execution.
- **Frontend**: Next.js and Tailwind for the IDE interface.
- **Collaboration**: Real-time multi-user editing with integrated whiteboarding.
- **Impact**: Achieved an 18% paid conversion rate during pilot phases by focusing on professional-grade tools for beginners.
